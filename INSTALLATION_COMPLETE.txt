## 1️⃣ Preparar Base de Datos (Gratis - Supabase)
────────────────────────────────────────────────
1. Ir a: https://supabase.com
2. Crear cuenta (gratis)
3. Crear nuevo proyecto
4. En SQL Editor → New Query
5. Abrir archivo: App Trading/backend/SQL_COPY_PASTE.txt
6. Copiar TODO el SQL (desde "-- Trading Opportunity..." hasta el final)
7. Pegar en SQL Editor de Supabase
8. Click "Run"
9. Esperar a que diga "Query Executed Successfully"
10. Copiar credenciales:
    - Project URL
    - Anon Key

PASO 2: Configurar Backend
──────────────────────────
1. Editar: backend/.env
   SUPABASE_URL=tu-url-aqui
   SUPABASE_KEY=tu-key-aqui
   DEBUG=True

PASO 3: Instalar y Ejecutar
───────────────────────────
Windows:
  setup.bat      # Primera vez para instalar dependencias
  run.bat        # Ejecutar la aplicación

macOS/Linux:
  bash setup.sh  # Primera vez para instalar dependencias
  bash run.sh    # Ejecutar la aplicación

PASO 4: Acceder
──────────────
Frontend:  http://localhost:3000
Backend:   http://localhost:8000
Docs API:  http://localhost:8000/docs (Swagger)

================================================================================
❓ SOPORTE Y TROUBLESHOOTING
================================================================================

Problema: "Error connecting to Supabase"
Solución:
  - Verificar SUPABASE_URL está correcto
  - Verificar SUPABASE_KEY está correcto
  - Verificar base de datos está inicializada

Problema: "Frontend no ve Backend"
Solución:
  - Verificar backend está corriendo en localhost:8000
  - Verificar en frontend/src/services/api.ts la URL

Problema: "No hay oportunidades detectadas"
Solución:
  - Esto es normal al principio
  - Necesita haber activos agregados
  - Necesita haber datos de mercado (se descargan auto)
  - Esperar 30 segundos para que se procesen datos

Problema: "Error al instalar dependencias"
Solución:
  - Verificar Python 3.9+ instalado
  - Verificar Node.js 16+ instalado
  - Ejecutar: python verify_installation.py

Para más ayuda, revisa los logs:
  - Backend: http://localhost:8000/docs
  - Frontend: Console del navegador (F12)
